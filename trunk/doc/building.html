System Prerequisites

[Contents]

  1. To reduce the effort needed to distribute HPCToolkit, we have opted to distribute it in source form only. You will need a version of CVS on your PATH.

  2. HPCToolkit requires an ANSI/ISO C/C++ compliant compiler. Generally speaking any relatively recent vendor compiler or GCC 3.x should be fine. The toolkit uses GNU binutils which requires a C compiler that also supports a few GNU extensions; obviously, GCC may be used.

    • IRIX64-mips:
      • SGI MIPSpro C++/C 7.3.x, 7.4
      • GCC C++/C 3.2.1
    • Linux-i686
      • GCC C++/C 3.0.4, 3.2.x, 3.3.x
    • Linux-ia64:
      • Intel C++/C 8.0
      • Intel C++/C 7.0, 7.1
      • Intel C++/C 6.0
      • GCC C++/C 3.0.4, 3.2.x, 3.3.x
    • Linux-opteron:
      • GCC C++/C 3.3.x
    • SunOS-sparc:
      • Sun Forte C++/C 5.3 (WorkShop 6 update 2 with patch 111685-01 or greater (bug id 4466915))
      • Sun Forte C++/C 5.1 (WorkShop 6)
      • GCC C++/C 3.0.4
    • Tru64-alpha:
      • Compaq C++/C 6.2, 6.3, 6.5
      • GCC C++/C 3.2.1

  3. hpcrun, a tool for collecting performance data on Linux, requires an installation of PAPI 3.0 or greater. To learn more about installing PAPI on your system consult our PAPI notes.

  4. hpcviewer is a Java application that can be used on a variety of platforms. It only needs a recent Java runtime environment (JRE). We have successfully used:

    • Mac OS X: JRE 1.4.2
    • Windows: JRE 1.4.2
    • Cygwin: JRE 1.4.2

    • Linux (i686, ia64, opteron): JRE 1.4.2
    • Solaris-sparc: JRE 1.4.0
    • Tru64-alpha: JRE 1.4.2

  5. HPCToolkit requires a basic version of GNU make, not a vendor supplied make. We have successfully used versions 3.70, 3.76, 3.77, 3.78, 3.79 3.80.

  6. HPCToolkit contains some Perl-based tools which require that some variant of Perl 5 installed and that it is in your path. We have succssfully used versions 5.6.x, 5.8.x.

Downloading the Sources

[Contents]

  1. Download the HPCToolkitRoot tarball and unzip/untar it.

    gunzip HPCToolkitRoot.tar.gz
    tar xvf HPCToolkitRoot.tar

  2. Change to the top-level HPCToolkitRoot directory.

  3. Gather the rest of the sources using our checkout tool. If you only want to download hpcviewer, use the --mod-hpcviewer-only option. (We often use hpcviewer on laptops or desktops rather than on the system on which we collect performance measurements.)

    ./bin/updatesrc
    or
    ./bin/updatesrc --mod-hpcviewer-only

    This will print a summary of progress to standard out and generate a log file with all the hoary details.

(Note: If you download HPCToolkit, it will also download three external libraries -- OpenAnalysis, Xerces-C++ and GNU binutils. In some cases it may be possible to use versions of these libraries that are already installed on your system. However, we do not support such configurations.)

Building and Installing

[Contents]

Temporary: To build HPCToolkit on IRIX64, please use these instructions.

The easiest way to build HPCToolkit is to use Makefile.quick, which will build everything you downloaded in the first step. If you want to build HPCToolkit's hpcrun, you will need to use the PAPI option. For a brief description, list of supported options and the default installation path, execute the following.

make -f Makefile.quick help

As a special exception, hpcviewer may be used without performing the steps below. This is useful in a non-Un*x environment (e.g. Windows without Cygwin).

  1. Change to the top-level HPCToolkitRoot directory.

  2. Configure, build and install. Since this will produce a lot of output, you may want to redirect it to a file.

    make -f Makefile.quick [PAPI=<install_path>] all

This will build and install HPCToolkit in the paths given above. Because the installation is compact and self-contained, you may rename or move it to another location. All source code may be removed.

Using

[Contents]

You may use the resulting installation as-is. However, you may wish to add the bin subdirectory to your PATH. One way to do this is to set the HPCTOOLKIT environment variable to the root of your installation and then use our 'Sourceme' script. For example:

(csh variants)
setenv HPCTOOLKIT "/my_path/HPCToolkit"
source $HPCTOOLKIT/Sourceme-csh

(sh variants)
export HPCTOOLKIT="/my_path/HPCToolkit"
source $HPCTOOLKIT/Sourceme-sh

As a special exception for non-Un*x environments, hpcviewer may be lauched via the GUI. See the man page for more information.
