System Prerequisites

[Contents]

  1. To reduce the effort needed to distribute HPCToolkit, we have opted to distribute it in source form only.

  2. HPCToolkit requires an ANSI/ISO C/C++ compliant compiler. Generally speaking any relatively recent vendor compiler or GCC 3.x and 4.x should be fine. The toolkit uses GNU binutils which requires a C compiler that also supports a few GNU extensions; obviously, GCC may be used.

    • Linux-x86
      • Intel C++/C 8.x, 9.x
      • GCC C++/C 3.x, 4.x
    • Linux-ia64:
      • Intel C++/C 8.x, 9.x
      • GCC C++/C 3.x, 4.x
    • Linux-x86_64:
      • Intel C++/C 9.x, 10.x
      • GCC C++/C 3.x, 4.x
      • Compiler bug: PGI 6.0.2, 6.1.2 (internal compiler error)

  3. hpcrun, a tool for collecting performance data on Linux, requires an installation of PAPI 3.0 or greater. To learn more about installing PAPI on your system consult our PAPI notes.

  4. hpcviewer is a Rich Client application that can be used on a variety of platforms. It needs a Java 1.5 runtime environment (JRE) and gtk (on Linux).

  5. HPCToolkit requires a basic version of GNU make, not a vendor supplied make. We have successfully used versions 3.70, 3.76, 3.77, 3.78, 3.79 3.80.

Downloading the Sources

[Contents]

  1. Download a HPCToolkit one-stop-shopping tarball and untar it:

    gunzip hpctoolkit-one-stop-shopping-x.y.z.tar.gz
    tar xvf hpctoolkit-one-stop-shopping-x.y.z.tar

(Note: The one-stop-shopping tarball contains sevearl external libraries, e.g. OpenAnalysis, Xerces-C++ and GNU binutils. In some cases it may be possible to use versions of these libraries that are already installed on your system. However, we do not support such configurations.)

Building and Installing

[Contents]

The easiest way to build HPCToolkit is to use Makefile.quick. If you want to build HPCToolkit's hpcrun, you will need to use the PAPI option. For a brief description, list of supported options and the default installation path, execute the following.

make -f Makefile.quick help

As a special exception, hpcviewer may be used without performing the steps below. This is useful in a non-Un*x environment (e.g. Windows without Cygwin).

  1. Change to the top-level hpctoolkit-one-stop-shopping-x.y.z directory.

  2. Configure, build and install. Since this will produce a lot of output, you may want to redirect it to a file. Make sure to use the CXX and CC options if you do not wish to use the GNU compiler:

    make -f Makefile.quick [PAPI=<install_path>] [CXX=<C++ compiler>] [CC=<C compiler>] all

    For example:

    • Linux-*: (using the GNU compiler and Intel compiler, respectively)
      • make -f Makefile.quick PAPI=<papi> CXX=g++ CC=gcc all
      • make -f Makefile.quick PAPI=<papi> CXX=icc CC=icc all

This will build and install HPCToolkit in the paths given above. Because the installation is compact and self-contained, you may rename or move it to another location. All source code may be removed.

Using

[Contents]

You may use the resulting installation as-is. However, you may wish to add the bin subdirectory to your PATH. One way to do this is to set the HPCTOOLKIT environment variable to the root of your installation and then use our 'Sourceme' script. For example:

(csh variants)
setenv HPCTOOLKIT "/my_path/HPCToolkit"
source $HPCTOOLKIT/Sourceme-csh

(sh variants)
export HPCTOOLKIT="/my_path/HPCToolkit"
source $HPCTOOLKIT/Sourceme-sh

As a special exception for non-Un*x environments, hpcviewer may be lauched via the GUI. See the man page for more information.
