User Commands: hpcquick(1)

SYNOPSIS

hpcquick [other-options] -z|x [ -I dir1 dir2 ... dirN ] [ -S struct1 struct2 ... structJ ] [ -G group1 group2 ... groupK ] -P prof1 prof2 ... profM

Location: ${HPCTOOLKIT}/bin

DESCRIPTION

hpcquick is a script that simplifies the process of using hpcview to correlate performance data with an application's source code. Given a list of paths to directories containing interesting program source code files, an optional list of program structure files created by the bloop tool, and a list of profile files, creates an hpcview configuration file and a browsable database in the current directory. All shell commands are captured into a log file.

Following is a list of the various profile files accepted. If needed, they will automatically be converted into the XML PROFILE format before hpcview is run.

  • binary output from hpcrun
  • binary output from SGI's ssrun
  • XML PROFILE files (e.g., from xprof, hpcprof and ptran

ARGUMENTS

-z
configure hpcview to produce an HTML database
-x
configure hpcview to produce an XML database
-I dir1 dir2 ... dirN
dir1 through dirN specify a list of directories -- either relative or absolute -- containing source code to which performance data should be correlated. In order to search for source files in a directory and recursively inspect all of its descendents, append an escaped '*' after the last slash, e.g., /mypath/\*. For any functions whose source code is not found in these directories (e.g., system libraries), hpcview will generate a synopsis that shows the presence of the function and its line extents (if known), but no source code.
-S struct1 struct2 ... structJ
struct1 through structJ is a list of program structure files (XML document of type PGM) that each contain a specification of the hierarchical structure of a binary image. The structure files can be for the main program and/or any or all of the shared libraries used by the program. Supplying program structure files is optional. However, an appropriate structure file must be supplied in order for hpcview to compute and correlate loop-level performance metrics.

Structure files are typically created using bloop to extract the structure information from the executable binary.
-S group1 group2 ... groupK
group1 through groupK is a list of group files (XML document of type PGM) that each define a partition (grouping) of the program structure scope tree. Group files typically closely correspond to structure files -- indeed they can replace structure files -- where the the groupings serve as handles that will be annotated with aggregate performance information by hpcview.
-P prof1 prof2 ... profM
prof1 through profM are names of profile files.

Other options

-n
No hpcview output. Convert profiles and create an hpcview config file, but do not generate an hpcview output dababase; record all commands in log file.
-V
Print version information.
-h
Print help.

EXAMPLES

  1. Assume all of your source code is in ../src and you did several ssrun executions in which you gathered hardware performance counter data (experiments named "*_hwc"). Program structure information from bloop has been sent to bloop.psxml. The command

    hpcquick -x -I ../src -S bloop.psxml -P *_hwc.*

    will display the profile data with one column per profile file.

NOTES

hpcquick runs on any system with a relatively recent version of Perl.
