User Commands: hpcquick(1)

SYNOPSIS

hpcquick [-V] -I dir1 dir2 ... dirN [-S structure] -P binprof1 binprof2 ... binprofM

Location: ${HPCTOOLKIT}/bin

DESCRIPTION

hpcquick is a script that simplifies the process of using hpcview to correlate performance data with an application's source code. Given a list of paths to directories containing interesting program source code files, an optional program structure file created by the bloop tool, and a list of binary profiles (collected using ssrun on the SGI/MIPS platform, or using uprofile on the Compaq/Alpha platform), hpcquick will run the vendor prof tool on the binary profile files and translate them into a platform-independent format using the ptran script. hpcquick will have hpcview generate a performance database in the HPCQUICK subdirectory. The configuration file synthesized by hpcquick and passed to hpcview will be left in a file hpcquick.xml in the current directory. A script containing the commands issued by hpcquick to create the performance database will be left in the file hpcquick.log in the current directory.

ARGUMENTS

-V
Print version information.
-I dir1 dir2 ... dirN
dir1 through dirN specify a list of directories -- either relative or absolute -- containing source code to which performance data should be correlated. In order to search for source files in a directory and recursively inspect all of its descendents, append an escaped '*' after the last slash, e.g., /mypath/\*. For any functions whose source code is not found in these directories (e.g., system libraries), hpcview will generate a synopsis that shows the presence of the function and its line extents (if known), but no source code.
-S structure
structure is a program structure file (XML document of type PGM) that contains a specification of the hierarchical structure of the application program. Supplying a program structure file is optional. However, a structure file must be supplied for hpcview to compute and correlate loop-level performance metrics.
-P binprof1 binprof2 ... binprofM
binprof1 through binprofM are names of binary profile files collected using SGI's ssrun, or the Compaq Tru64's uprofile command. Given the names of one or more profiles, hpcquick will translate them into vendor-specific ASCII representations using the vendor-supplied prof tool, and then translate them into a platform-independent representation (XML documents of type PROFILE) for input to hpcview.

Structure files are typically created using bloop to extract the structure information from the executable binary. (However, other mechanisms for creating a structure file could be used.)

EXAMPLES

  1. Assume all of your source code is in ../src and you did several ssrun executions in which you gathered hardware performance counter data (experiments named "*_hwc"). Program structure information from bloop has been sent to bloop.psxml. The command

    hpcquick -I ../src -S bloop.psxml -P *_hwc.*

    will display the profile data with one column per profile file.

NOTES

hpcquick runs on any system with a relatively recent version of Perl. Depending on the version of Perl on your system, either the PERLLIB or PERL5LIB environment variable must be set to point to the Perl library located in ${HPCTOOLKIT}/bin/perl. The Sourceme-csh and Sourceme-sh files set both of these variables for csh-based and sh-based shells, respectivley.
