User Commands: hpcquick(1)

SYNOPSIS

hpcquick [-V] -z|x [-n] -I dir1 dir2 ... dirN [-S structure] -P prof1 prof2 ... profM

Location: ${HPCTOOLKIT}/bin

DESCRIPTION

hpcquick is a script that simplifies the process of using hpcview to correlate performance data with an application's source code. Given a list of paths to directories containing interesting program source code files, an optional program structure file created by the bloop tool, and a list of profile files, creates an hpcview configuration file and a browsable database in the current directory. All shell commands are captured into a log file.

Following is a list of the various profile files accepted. If needed, they will automatically be converted into the XML PROFILE format (using the vendor's prof tool and the toolkit's ptran) before hpcview is run.

  • binary output from SGI's ssrun
  • XML PROFILE files from xprof, papiprof and ptran

ARGUMENTS

-V
Print version information.
-z
configure hpcview to produce an HTML database
-x
configure hpcview to produce an XML database
-n
exit before producing database (but record command in log file)
-I dir1 dir2 ... dirN
dir1 through dirN specify a list of directories -- either relative or absolute -- containing source code to which performance data should be correlated. In order to search for source files in a directory and recursively inspect all of its descendents, append an escaped '*' after the last slash, e.g., /mypath/\*. For any functions whose source code is not found in these directories (e.g., system libraries), hpcview will generate a synopsis that shows the presence of the function and its line extents (if known), but no source code.
-S structure
structure is a program structure file (XML document of type PGM) that contains a specification of the hierarchical structure of the application program. Supplying a program structure file is optional. However, a structure file must be supplied for hpcview to compute and correlate loop-level performance metrics.

Structure files are typically created using bloop to extract the structure information from the executable binary.
-P prof1 prof2 ... profM
prof1 through profM are names of profile files.

EXAMPLES

  1. Assume all of your source code is in ../src and you did several ssrun executions in which you gathered hardware performance counter data (experiments named "*_hwc"). Program structure information from bloop has been sent to bloop.psxml. The command

    hpcquick -I ../src -S bloop.psxml -P *_hwc.*

    will display the profile data with one column per profile file.

NOTES

hpcquick runs on any system with a relatively recent version of Perl. Depending on the version of Perl on your system, either the PERLLIB or PERL5LIB environment variable must be set to point to the Perl library located in ${HPCTOOLKIT}/lib/perl. The Sourceme-csh and Sourceme-sh files set both of these variables for csh-based and sh-based shells, respectivley.
