User Commands: ptran(1)

SYNOPSIS

ptran [-V] [-m platform] [-e nn] [-h] < input_file > output_file.pxml

Location: ${HPCTOOLKIT}/bin

DESCRIPTION

ptran is a Perl script that reads a vendor-specific profile file and outputs a PROFILE document. Each profile data file provided to hpcview must be in this format.

ARGUMENTS

-V
Print version information.
-m platform
platforms currently supported are sgi and compaq unless specified explicitly, the platform name is inferred using the output from uname(1).
-e nn
nn is an integer that selects specific metrics from experiments (e.g. ideal on SGIs) that generate more than one interesting column in the vendor-format prof file.
input_file
The current release includes ptran scripts for interpeting vendor-specific profile information on SGI and Compaq platforms.

On an SGI system, apply prof -lines to an output file generated by ssrun.

On a Compaq Tru64/Alpha system, apply prof -lines to a umon.out file generated by uprofile.
-h
Print help.

EXAMPLES

  1. To send PROFILE information to the file fcy_hwc.pxml for the program foo using the ssrun experiment named `fcy_hwc', use the following commands:

    ssrun -fcy_hwc foo
    prof -lines foo.fcy_hwc.m* | ptran > fcy_hwc.pxml

  2. To send PROFILE information to the file cycles.pxml for the program foo using the uprofile experiment named `cycles0', use the following commands:

    uprofile cycles0 foo
    prof -lines foo umon.out > foo.cycles
    ptran < foo.cycles > cycles.pxml

NOTES

ptran runs on any system with a relatively recent version of Perl. Depending on the version of Perl on your system, either the PERLLIB or PERL5LIB environment variable must be set to point to the Perl library located in ${HPCTOOLKIT}/lib/perl. The Sourceme-csh and Sourceme-sh files set both of these variables for csh-based and sh-based shells, respectively.
